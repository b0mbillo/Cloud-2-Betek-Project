Preguntas Frecuentes

¿Cómo funciona la integración con AWS? ▼
Nuestro sistema está construido en AWS utilizando:
- API Gateway: Enrutamiento inteligente de solicitudes
- Lambdas: Funciones serverless para cada operación (login, crear ticket, gestionar tickets)
- JWT: Autenticación segura entre cliente y backend
Puedes consultar el archivo api-config.js para configurar los endpoints.
¿Cómo configuro los endpoints del API? ▼
Sí, puedes adjuntar archivos (capturas de pantalla, logs, etc.) con un tamaño máximo de 5MB por archivo. Simplemente usa el área de drag & drop en el formulario de creación de tickets.
¿Qué endpoints Lambda tengo que crear? ▼
Necesitas crear Lambdas para estas operaciones:
1. POST /auth/login - Autenticar usuario y devolver JWT
2. POST /tickets/create - Crear nuevo ticket
3. GET /tickets/list - Listar todos los tickets
4. GET /tickets/:id - Obtener ticket por ID
5. PUT /tickets/:id/update - Actualizar ticket
6. DELETE /tickets/:id/delete - Eliminar ticket
¿Cómo manejo la autenticación en el frontend? ▼
La autenticación se realiza automáticamente:
1. El usuario hace login a través del formulario
2. El Lambda de autenticación valida credenciales y devuelve un JWT
3. El token se almacena en localStorage como authToken
4. Todas las solicitudes posteriores incluyen el token en el header Authorization: Bearer {token}
5. Si el token expira, se redirige automáticamente al login
¿Cómo depliego las Lambdas en AWS? ▼
Pasos básicos para desplegar:
1. Crea una función Lambda para cada endpoint en AWS Console
2. Configura el API Gateway para rutear las solicitudes
3. Asegúrate de que el Lambda devuelve JSON con estructura consistente
4. Habilita CORS en el API Gateway para solicitudes desde el frontend
5. Actualiza API_CONFIG.BASE_URL en api-config.js con tu URL de API Gateway
